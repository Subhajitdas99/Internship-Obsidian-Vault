# Submission Checklist

> **Use this checklist** before submitting your Week-1 deliverable

---

## Required Deliverables

### 1. GitHub Repository

- [ ] Repository is **PUBLIC** (we need to see it)
- [ ] Repository has a **clear name** (e.g., `ai-modernization-tool`, `code-analyzer`)
- [ ] Repository URL shared in Teams/Slack

### 2. README.md

- [ ] **What you built**: One paragraph explaining the tool
- [ ] **How to run it**: Clear instructions
- [ ] **What you learned**: Key insights
- [ ] **Challenges**: What was hard?

### 3. Working Code

- [ ] Code **runs without errors** (or errors are documented)
- [ ] Code **produces output** (JSON, diagrams, or text)
- [ ] Code is **readable** (comments where needed)
- [ ] No hardcoded paths (or documented)

### 4. Output Artifacts

At least one of:
- [ ] `entities.json` - Extracted classes/functions
- [ ] `relationships.mermaid` - Call graph or dependency diagram
- [ ] `summary.md` - What you found, explained

### 5. Loom Video (5-10 minutes)

- [ ] **Introduction**: Your name, chosen project
- [ ] **Problem explanation**: What are you solving?
- [ ] **Demo**: Show your tool working
- [ ] **Learnings**: What did you learn?
- [ ] **Challenges**: What was difficult?
- [ ] Video URL shared in Teams/Slack

### 6. REFLECTION.md (Optional but Recommended)

- [ ] What was your approach?
- [ ] What would you do differently?
- [ ] What questions remain?

## What We're Looking For

### Green Flags (Good Signs)

- Clear explanation of approach
- Code that demonstrates understanding
- Honest reflection on challenges
- Initiative beyond minimum requirements
- Evidence of learning

### Red Flags (Warning Signs)

- No explanation, just code
- Code that doesn't run
- No output artifacts
- No video or very short video
- Copy-pasted without understanding

## Common Mistakes to Avoid

| Mistake | How to Avoid |
|---------|--------------|
| **Private repo** | Check settings â†’ Public |
| **No instructions** | Test README on a fresh clone |
| **Hardcoded paths** | Use relative paths or config |
| **No output** | Verify tool produces files |
| **Silent failures** | Add error messages |
| **Too long video** | Edit to 10 min max |
| **No audio** | Test before recording |

## Submission Process

1. **Push all code** to your public GitHub repo
2. **Create README.md** with instructions
3. **Generate outputs** and commit them
4. **Record Loom video** and get shareable link
5. **Post in Teams/Slack**:
   ```
   Name: [Your Name]
   Project: [ERPNext/OpenElis/Bahmni]
   GitHub: [URL]
   Loom: [URL]
   Brief description: [1-2 sentences]
   ```
