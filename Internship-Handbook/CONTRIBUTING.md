## üîÑ Contribution Process

### Step 1: Identify What to Contribute
- Found an error? Fix it!
- Learned something useful? Share it!
- Solved a problem? Document it!
- Have a better way? Suggest it!

### Step 2: Make Your Changes

#### For Small Changes (Typos, Quick Fixes)
1. Edit the file directly in Obsidian
2. Add a brief note about what you changed
3. Save the file

#### For Larger Contributions
1. Create a new branch in Git
2. Make your changes
3. Test that links work
4. Submit a pull request

### Step 3: Follow the Format

#### File Naming
- Use kebab-case: `my-new-document.md`
- Be descriptive: `git-workflow-guide.md` not `guide.md`
- Keep it concise but clear

#### Markdown Standards
```markdown
# Main Title

## Section Heading

### Subsection

- Bullet points for lists
- Use consistent formatting

1. Numbered lists for steps
2. Keep numbering sequential

**Bold** for emphasis
*Italic* for subtle emphasis
`code` for inline code

```language
Code blocks with language specified
```

| Tables | Are | Useful |
|--------|-----|--------|
| Data | Here | Clear |

[[Internal-Link|Display Text]]
[External Link](https://example.com)
```

### Step 4: Add Metadata

At the bottom of each document, add:
```markdown
---
*Last Updated: [[YYYY-MM-DD]]*
*Contributors: [Your Name]*
*Category: [Handbook/Programme/Support/Teams/Resources]*
```

## üìö Creating New Sections

### When to Create New Content
- Topic not covered in existing docs

```markdown
